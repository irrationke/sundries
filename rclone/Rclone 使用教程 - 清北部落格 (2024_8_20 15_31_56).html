Rclone 使用教程

1、關於

介紹

Rclone 是一個開源、多執行緒、命令列介面的電腦程式，可用於管理雲端儲存。 其功能包括檔案同步、文件傳輸、加密、快取和掛載。
Rclone支援包括 Amazon S3 和 Google 雲端硬碟在內的共五十多種雲端儲存服務。
——Wikipedia

詳細介紹可以檢視 Rclone 官網：https://rclone.org/。

功能

Rclone 支援以下功能：

  • 將檔案備份（和加密）到雲端儲存
  • 從雲端儲存恢復（和解密）檔案
  • 將雲資料映象到其他雲服務或本地
  • 將資料遷移到雲端，或在雲端儲存供應商之間遷移
  • 將多個、加密、快取或多樣化的雲端儲存掛載為磁碟
  • 使用lsf、ljson、size、ncdu分析和說明雲端儲存上儲存的資料
  • 聯合檔案系統一起呈現多個本地和/或雲檔案系統作為一個

特性

Rclone 具有以下特性：

  • 接送服務
  • 始終檢查 MD5、SHA1 雜湊值以確保檔案完整性
  • 時間戳保留在檔案中
  • 可以隨時重新啟動操作
  • 可以進出網路，例如兩個不同的雲提供商
  • 可以使用多執行緒下載到本地磁碟
  • 將新檔案或更改的檔案複製到雲端儲存
  • 同步（一種方式）使目錄相同
  • 驗證後將檔案移動到雲端儲存刪除本地
  • 檢查雜湊和丟失/額外的檔案
  • 將您的雲端儲存掛載為網路磁碟
  • 透過 HTTP / WebDav / FTP / SFTP / DLNA 服務本地或遠端檔案
  • 實驗性基於 Web 的 GUI

2、安裝

Rclone 是單檔案程式，可以直接下載最新 Release，設定環境變數後即可使用。

如果伺服器是 ARM 或其它架構，可以自行查詢官網並替換下載連結。

3、配置

配置有兩種方法：

  1. 編輯 /root/.config/rclone/rclone.conf ，追加新的內容。
  2. 執行 rclone config ，使用互動式配置。

互動式配置非常簡單易用，不過如果沒有特殊需求注意跳過長長的“高階設定（Advanced options）”，這裡面的配置可能會多達幾十項，配置錯誤可能導致無法使用，一般保持預設就好。

配置完成後可以直接複製配置檔案，在另一臺伺服器上就可以立即使用。

4.2、常用指令

使用 rclone --help 可檢視所有命令，這裡列出常用命令，其他的命令參考官方文件。

命令說明
rclone copy複製
rclone move移動；如果要在移動後刪除空源目錄，加上 --delete-empty-src-dirs 引數
rclone mount掛載
rclone sync（單向）同步：將源目錄同步到目標目錄，只更改目標目錄
rclone size檢視雲端檔案佔用大小
rclone delete刪除路徑下的檔案內容
rclone purge刪除路徑及其所有檔案內容
rclone mkdir建立目錄
rclone rmdir刪除目錄
rclone rmdirs刪除指定環境下的空目錄；如果加上 --leave-root 引數，則不會刪除根目錄
rclone check檢查源和目的地址資料是否匹配
rclone ls列出指定路徑下的所有的檔案以及檔案大小和路徑
rclone lsl比上面多一個顯示上傳時間
rclone lsd列出指定路徑下的目錄
rclone lsf列出指定路徑下的目錄和檔案

4.3、常用引數

執行

命令說明
-n = --dry-run測試執行，檢視 Rclone 在實際執行中會進行哪些操作
-P = --progress顯示實時傳輸進度，500ms 重新整理一次（如不指定預設1分鐘重新整理一次）
--cache-chunk-size 5M塊的大小，預設 5M，越大上傳越快，佔用記憶體越多，太大可能會導致程序中斷
--onedrive-chunk-size 100M提高 OneDrive 上傳速度，適用於大頻寬伺服器
--drive-chunk-size 64M提高 Google Drive 上傳速度，適用於大頻寬伺服器
--cache-chunk-total-size SizeSuffix塊可以在本地磁碟上佔用的總大小
--transfers=N並行檔案數，預設為4。在比較小的記憶體的伺服器上建議調小這個引數，比如 128M 的小雞上使用建議設定為1。
--config <path>指定<path>為配置檔案路徑
--ignore-errors跳過錯誤
--size-only根據檔案大小校驗，不校驗雜湊值
--drive-server-side-across-configs服務端對服務端傳輸

日誌

rclone 有 4 個級別的日誌記錄， ERROR ， NOTICE ， INFO 和 DEBUG 。預設情況下，rclone 將生成 ERROR 和 NOTICE 級別訊息。

引數說明
-qrclone 將僅生成 ERROR 訊息
-vrclone 將生成 ERROR，NOTICE 和 INFO 訊息，推薦此項
-vvrclone 將生成 ERROR，NOTICE，INFO 和 DEBUG 訊息
--log-level LEVEL標誌控制日誌級別

過濾

同步時有些檔案不需要同步，可以使用過濾規則排除。

命令說明
--exclude-from排除檔案或目錄列表來自某個檔案指定
--include-from包含檔案或目錄來自某個檔案指定
--filter-from檔案過濾規則來自某個檔案指定
--exclude排除檔案或目錄
--include包含檔案或目錄
--filter檔案過濾規則。
包含規則以+開頭，排除規則以-開頭

目錄過濾

目錄過濾需要在目錄名稱後面加上 / ，否則會被當做檔案進行匹配。以 / 開頭只會匹配根目錄（指定目錄下），否則匹配所目錄。這同樣適用於檔案。

命令說明
--exclude ".git/"排除所有目錄下的 .git 目錄
--exclude "/.git/"只排除根目錄下的 .git 目錄
--exclude "{Video,Software}/"排除所有目錄下的 Video 和 Software 目錄
--exclude "/{Video,Software}/"只排除根目錄下的 Video 和 Software 目錄
--include "/{Video,Software}/**"僅包含根目錄下的 Video 和 Software 目錄的所有內容

大小過濾

預設大小單位為 kBytes ，但可以使用 k ， M 或 G 字尾。

命令說明
--min-size過濾小於指定大小的檔案。比如 --min-size 50 表示不會傳輸小於 50k 的檔案。
--max-size過濾大於指定大小的檔案。比如 --max-size 1G 表示不會傳輸大於 1G 的檔案。

使用配置檔案過濾

命令說明
--exclude-from <規則檔案>從檔案新增包含規則
--include-from <規則檔案>從檔案新增刪除規則
--filter-from <規則檔案>從檔案新增包含 / 排除規則。比如 --filter-from filter-file.txt

4.4、環境變數

Rclone 中的每個選項都可以透過環境變數設定。環境變數的名稱可以透過長選項名稱進行轉換，刪除 -- 字首，更改 - 為 _ ，大寫並新增字首 RCLONE_ 。

環境變數的優先順序低於命令列選項，即透過命令列追加的選項會覆蓋環境變數的設定。

命令說明
RCLONE_CONFIG自定義配置檔案路徑
RCLONE_CONFIG_PASS若 Rclone 進行了加密設定，把此環境變數設定為密碼，可自動解密配置檔案
RCLONE_RETRIES上傳失敗重試次數，預設 3 次
RCLONE_RETRIES_SLEEP上傳失敗重試等待時間，預設禁用，單位s、m、h分別代表秒、分鐘、小時
CLONE_TRANSFERS並行上傳檔案數
RCLONE_CACHE_CHUNK_SIZE塊的大小
RCLONE_CACHE_CHUNK_TOTAL_SIZE塊可以在本地磁碟上佔用的總大小
RCLONE_IGNORE_ERRORS=true跳過錯誤
